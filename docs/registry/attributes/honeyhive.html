HoneyHive Config Bucket Attributes ​

Sub-keys of the config bucket that identify the model + provider and drive Template-section rendering with substitution. All attributes in this group are stable.

Attributes:

KeyStabilityValue TypeDescriptionExample Values
honeyhive.config.modelstringModel identifier used for the call. Read by the Playground when loading an event for replay; also used by the Trajectory view as a category label fallback.gpt-4o; claude-3-5-sonnet-20241022; gemini-1.5-pro
honeyhive.config.providerstringProvider name. Read by the Playground when loading an event for replay; mirrors metadata.provider (set from the source gen_ai.system attribute or its v1.37+ gen_ai.provider.name successor).openai; anthropic; bedrock; google
honeyhive.config.templatestringPrompt template. When set as an array of template message objects (each with role and content), the sideview renders a Template section above the chat history and substitutes placeholders with matching keys from inputs (non-chat-history inputs); template.length also controls how many leading messages in inputs.chat_history are skipped by the main chat renderer. May alternatively be an object with name, version, and content sub-keys. Stored as JSON when transported via OTLP.[{"role":"system","content":"You are an assistant. User name: ."}]; {"name":"customer_support_v2","version":"3","content":"..."}

HoneyHive Inputs Bucket Attributes ​

Sub-keys of the inputs bucket that drive special platform rendering (Chat History tab, template substitution). All attributes in this group are stable.

Attributes:

KeyStabilityValue TypeDescriptionExample Values
honeyhive.inputs.chat_historystringMessage thread array. Triggers the dedicated Chat History tab in the Input section of the sideview (excluded from the general Inputs key-value view); rendered via OpenAIChatRenderer. When config.template is also present, the first template.length messages are rendered as template messages with substitution. Expected format: array of message objects with at minimum a role field and one of content, value, tool_calls, or function_call. Stored as JSON when transported via OTLP.[{"role":"user","content":"Hello"},{"role":"assistant","content":"Hi there!"}]

HoneyHive Metadata Bucket — Session-Level Aggregates ​

Session-level aggregate sub-keys of the metadata bucket. Set on events where event_type == "session" by the writer/normalizer (rolled up from child events); not user-set on non-session events. Source for the Session Summary panel stats and the sessions table columns. All attributes in this group are stable.

Attributes:

KeyStabilityValue TypeDescriptionExample Values
honeyhive.metadata.costdoubleAggregate cost in USD attributed to this session (rolled up across child spans by the normalizer). Displayed as the "Cost" stat in the Session Summary panel (formatted as $X.XXXX) and as the built-in Cost sizing option in the Trajectory view.0.0123; 1.4521
honeyhive.metadata.has_feedbackbooleanTrue when the session has at least one feedback annotation across its events.true; false
honeyhive.metadata.num_eventsintTotal child event count for the session. Displayed as the "Num of Events" column in the sessions table and the "Number of children" stat in Session Summary. Initialized to 0 by the normalizer on session events.12
honeyhive.metadata.num_model_eventsintModel event count (children with event_type == "model"). Displayed as the "Num of LLM Requests" column in the sessions table. Initialized to 0 by the normalizer on session events.4

HoneyHive Metadata Bucket — Finish Reasons ​

Finish-reason sub-keys of the metadata bucket. The scalar finish_reason is extracted as the first element of the source gen_ai.response.finish_reasons array; the full array is preserved in finish_reasons for multi-choice responses. Mixed stability: finish_reason and finish_reasons are stable; the response_finish_reasons OpenInference alias is development-tier.

Attributes:

KeyStabilityValue TypeDescriptionExample Values
honeyhive.metadata.finish_reasonstringScalar model finish reason. Extracted as the first element of the source gen_ai.response.finish_reasons array (OTel GenAI, Traceloop) or set directly from gen_ai.response.finish_reason (OpenInference singular form).stop; length; tool_calls; content_filter
honeyhive.metadata.finish_reasonsstring[]Array of all model finish reasons (one entry per response choice). Mapped from gen_ai.response.finish_reasons across OTel GenAI, OpenInference, and Traceloop.["stop"]; ["stop", "tool_calls"]
honeyhive.metadata.response_finish_reasonsstring[]Array of model finish reasons set by the OpenInference ingestion strategy alongside metadata.finish_reasons for backwards compatibility with older OpenInference versions that emitted the attribute under a response_* prefix. Both keys are stored when the source attribute is present. [1]["stop"]; ["length", "tool_calls"]

[1] honeyhive.metadata.response_finish_reasons: Prefer metadata.finish_reasons (stable) for new code paths. This alias may be removed in a future minor release once OpenInference instrumentation has aged out.

HoneyHive Metadata Bucket — Identity Attributes ​

Agent and model identity sub-keys of the metadata bucket. Drive the Trajectory view category labels and the resolved-model display in the sideview. Mixed stability: the explicitly-stable sub-keys (agent_name, agent_id, tool_name, model_name, provider) are populated by the ingestion normalizer from OTel GenAI, OpenInference, and Traceloop; development-tier complements cover span classification, alias keys, and informational copies of root event fields.

Attributes:

KeyStabilityValue TypeDescriptionExample Values
honeyhive.metadata.agent_descriptionstringFree-text description of the agent that produced the span.A helpful research assistant that can analyze information and provide insights; An agent that provides assistance with ability to use tools.
honeyhive.metadata.agent_idstringAgent identifier. Mapped from gen_ai.agent.id across OTel GenAI, OpenInference, and Traceloop.agent-7c1f9d34
honeyhive.metadata.agent_namestringAgent name. Read by the Trajectory view as the primary per-row category label (first non-empty wins, falling back to tool_name, config.model, then event_name).planning_agent; router
honeyhive.metadata.conversation_idstringProvider-side conversation identifier, normalized from gen_ai.conversation.id (OTel) or session.id (OpenInference). [2]conv-7f3a9c
honeyhive.metadata.event_idstringInternal copy of an upstream event identifier — typically the MCP/tool-call ID or a Google ADK event ID — set into metadata alongside the root event_id. [3]tool_call_a1b2c3
honeyhive.metadata.event_namestringInformational copy of the event name label set by the ingestion normalizer alongside the root event_name. Does not override the root field.openai.chat.completion; retrieve_documents
honeyhive.metadata.event_typestringInformational copy of the event type label set by the ingestion normalizer alongside the root event_type. Does not override the root field.model; tool; agent
honeyhive.metadata.llm.model_namestringLegacy dotted alias for the resolved model name, set by older OpenInference instrumentation alongside metadata.model_name. Both keys are stored when the source attribute is present. [4]gpt-4o-mini; claude-3-5-sonnet-20241022
honeyhive.metadata.model_namestringResolved model name (response model when available, otherwise request model). The normalizer falls back to config.model when no response model is reported. Use metadata.response_model for the exact model returned by the API.gpt-4o-2024-08-06; claude-3-5-sonnet-20241022
honeyhive.metadata.operation_namestringGenAI operation name (chat, completion, embedding, execute_tool, …).chat; execute_tool; embedding
honeyhive.metadata.providerstringProvider name (mirrors config.provider). Set from the source gen_ai.provider.name (OTel v1.37+) or llm.provider (OpenInference, Traceloop). Distinct from metadata.system, which mirrors gen_ai.system.openai; anthropic
honeyhive.metadata.request_typestringLLM request type (chat, completion, embedding, …).chat; completion; embedding
honeyhive.metadata.span_kindstringInstrumentor span classification, normalized from openinference.span.kind, traceloop.span.kind, and (in OTel v1.37+) gen_ai.agent.type. Used as a fallback signal when mapping a span to the HoneyHive root event_type. [5]LLM; AGENT; TOOL
honeyhive.metadata.systemstringProvider system string mirroring config.provider (stable); set by the normalizer from gen_ai.system. [6]openai; anthropic; bedrock
honeyhive.metadata.tool_namestringTool name. Read by the Trajectory view as a category label fallback after agent_name. Mapped from gen_ai.tool.name across OTel GenAI, OpenInference, and Traceloop.web_search; sql_query_tool

[2] honeyhive.metadata.conversation_id: Does not override the HoneyHive root session_id field. metadata.conversation_id records the upstream concept; root session_id is the HoneyHive grouping key.

[3] honeyhive.metadata.event_id: Informational only. Does not override the root event_id field used to key the event in storage.

[4] honeyhive.metadata.llm.model_name: Prefer metadata.model_name (stable, HHAI-5104) for new code. This alias persists so historical events instrumented with older OpenInference versions remain queryable by the same key they were written with.

[5] honeyhive.metadata.span_kind: Values are stored in the casing emitted by the source instrumentor (OpenInference uppercases LLM/TOOL/AGENT; Traceloop lowercases llm/tool/agent). The enum captures the OpenInference casing as canonical members; extra string values are accepted.

[6] honeyhive.metadata.system: metadata.system and config.provider are populated from different source attributes. gen_ai.system (all three ecosystems) flows to metadata.system. gen_ai.provider.name (OTel v1.37+) and llm.provider (OpenInference, Traceloop) flow to config.provider.

honeyhive.metadata.span_kind has the following list of well-known values. If one of them applies, then the respective value MUST be used; otherwise, a custom value MAY be used.

ValueDescriptionStability

honeyhive.metadata.mcp_server_namestringMCP server name attached to the span by the MCP-aware instrumentation. [8]filesystem; github

[7] honeyhive.metadata.mcp_client_name: Routed into the metadata bucket. No corresponding stable attribute exists in OTel GenAI v1.36 or v1.37; HoneyHive normalizes from the OpenInference and standard-genai source attributes that carry the client name on the MCP-tool-call span.

[8] honeyhive.metadata.mcp_server_name: Routed into the metadata bucket. No stable cross-ecosystem source.

[9] honeyhive.metadata.mcp_session_id: Does not override the root session_id field used for trace grouping. Stored alongside it for protocol-level diagnostics.

[10] honeyhive.metadata.mcp_tool_name: Distinct from config.tool_name and metadata.tool_name — those carry the framework-level tool identity. mcp_tool_name carries the tool identifier as exposed by the MCP server.

HoneyHive Outputs Bucket Attributes ​

Sub-keys of the outputs bucket that select the chat-message rendering mode in the sideview, reconstruct flattened tool-call structures, and bypass bucket rendering for plain-text outputs. All attributes in this group are stable.

Attributes:

KeyStabilityValue TypeDescriptionExample Values
honeyhive.outputs.chat_historystringArray of output messages. When outputs.role is absent, the presence of this key still triggers chat-message rendering mode and the first element is rendered as the primary output. Stored as JSON when transported via OTLP; expected format matches inputs.chat_history.[{"role":"assistant","content":"The answer is 42."}]
honeyhive.outputs.rolestringSpeaker role for the output message. Presence of this key triggers the chat-message rendering mode in the sideview (capitalised as the speaker label). When absent, the output renders in the generic markdown/JSON toggle view.assistant; tool
honeyhive.outputs.textstringPlain-text output. When set, this string is used as the output value instead of the full outputs object; rendered in the standard markdown/JSON toggle view.The weather in San Francisco is 68F and sunny.
honeyhive.outputs.tool_callsstringArray of tool-call objects (each with id, function.name, function.arguments). When set as flattened dot-notation keys (tool_calls.0.id, tool_calls.0.function.name, ...), the sideview reconstructs a proper tool_calls array before rendering. Each tool call is rendered as a named JSON block. Stored as JSON when transported via OTLP.[{"id":"call_abc","function":{"name":"get_weather","arguments":"{\"city\":\"SF\"}"}}]
