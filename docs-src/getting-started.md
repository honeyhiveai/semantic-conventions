# Getting Started

Welcome to the HoneyHive Semantic Conventions documentation.

## What are Semantic Conventions?

Semantic conventions define a standard set of attribute names, types, and meanings for observability telemetry in AI applications. By adhering to these conventions, all HoneyHive SDKs and integrations maintain consistency in how they describe AI interactions, LLM calls, datasets, evaluations, and platform events.

## Using Semantic Conventions in Your SDK

When instrumenting your application with HoneyHive SDKs, the conventions are already baked in—SDK methods automatically populate attributes according to these standards. You rarely need to reference the conventions directly, but understanding them helps you interpret observability data in the HoneyHive platform and write more effective queries and evaluations.

For detailed attribute definitions and examples, browse the full Registry section. Each attribute includes its type, expected values, and usage context.
